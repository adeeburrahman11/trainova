<h1><a id="features"></a>Features</h1>

<table>
  <tr>
    <td width="50%" valign="top">

### Workout & Library
- Extensive Exercise Library & Muscle Mapping
- Favorite exercises for quick access
- Detailed instruction sets and tracking
- Custom built-in Rest & Interval Timers

</td>
    <td width="50%" valign="top">

### Adaptive Systems
- Quick-toggle Metric (kg/cm) and Imperial (lbs/in) conversion
- Precision user profiling (Age, Weight, Goal metrics)
- Real-time OTA App Updates connected directly to GitHub Releases
- In-App Feature Request communication portal

</td>
  </tr>
  <tr>
    <td width="50%" valign="top">

### Interface & Aesthetics
- Fluid, Adaptive Material 3 Interface
- Reactive Light Mode & Dark Mode
- Battery-saving AMOLED Black Mode 
- Dynamic contrast optimization based on environment

</td>
    <td width="50%" valign="top">

### Architecture & Privacy
- Completely Offline-First architecture (no internet required for core functions)
- Single-Device data persistence
- No hidden tracking or intrusive ad services
- Fully open and transparent source logic

<h1><a id="support-the-project"></a>Contributing & Support</h1>

### Trainova is completely free and open-source.

<div align="left">

We highly encourage contributions from the community. Whether you're adding new exercises to the database, fine-tuning the timer logic, or refining the UI rendering:

1. **Fork** the repository
2. **Create** your feature branch (`git checkout -b feature/NewExercise`)
3. **Commit** your changes (`git commit -m 'Add Squat variations'`)
4. **Push** to the branch (`git push origin feature/NewExercise`)
5. **Open a Pull Request**

If you encounter bugs, absolutely feel free to use the in-app **Feature Request** tool or drop an Issue on this repository directly.
